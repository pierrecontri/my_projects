Introduction

Ce Starter Kit Visual C# est une application d'écran de veille Windows prête à exécuter. L'écran de veille affiche une séquence d'images — soit des images par défaut, soit des images d'un répertoire que vous spécifiez — recouvertes de texte provenant d'un flux RSS (Really Simple Syndication) Internet en direct. Le projet est déjà prêt à être compilé et exécuté, et vous pouvez le personnaliser par le biais d'une programmation C# supplémentaire minime. Nous avons inclus une liste de suggestions dans la section Amélioration de l'écran de veille. Vous êtes également libre d'utiliser le code source comme base pour vos propres projets et de partager votre travail avec d'autres personnes ou de le télécharger sur Internet.


Remarque : Cette documentation suppose que vous disposez d'une connaissance de base des concepts de programmation et de l'environnement Visual C#. Pour en apprendre davantage sur ces rubriques, cliquez sur Aide dans le menu principal de la documentation du produit, puis sélectionnez Sommaire, Index ou Recherche. Vous pouvez également accéder à l'aide en plaçant le curseur de la souris sur des mots clés du langage ou des éléments de l'interface utilisateur tels que des fenêtres ou des boîtes de dialogue, et en appuyant sur F1.

Objectifs

Lorsque vous aurez lu la documentation de ce Starter Kit, vous saurez comment installer et configurer l'application Écran de veille, de quelle manière le code source est organisé, comment fonctionne l'application et comment la personnaliser à l'aide de Visual C#. Le Starter Kit illustre les technologies suivantes :

  • GDI+ — pour mettre en forme et afficher des graphismes et du texte
  • System.XML — utilisation de XmlTextReader et de XmlNode pour lire les fichiers XML contenant des informations sources RSS
  • La classe WebClient — pour télécharger des données à partir du Web
  • RSS — inclusion de flux, de canaux et d'éléments RSS
  • Le langage Visual C# 2.0 — inclusion de génériques

Mise en route

Génération et exécution de l'écran de veille

Une fois votre projet chargé dans l'environnement Visual C#, vous pouvez compiler et exécuter le programme en une seule opération.

Pour générer et exécuter l'application d'écran de veille

  • Appuyez sur F5 pour générer et démarrer l'écran de veille sous le débogueur.

L'écran de veille est lancé et il affiche les articles les plus récents provenant du flux RSS. Si vous appuyez sur une touche ou déplacez la souris, l'écran de veille se ferme.

Apport d'une modification rapide au code source

Il est facile de changer le code source pour modifier le fonctionnement du programme. Voici un exemple qui ajoute votre propre chaîne de texte à l'affichage de l'écran de veille.

Pour afficher une ligne de texte supplémentaire :

  1. Recherchez la fenêtre Explorateur de solutions ; elle se trouve par défaut dans le coin supérieur droit de l'environnement Visual C#. Cliquez avec le bouton droit sur ScreenSaverForm.cs dans la fenêtre Explorateur de solutions, puis sélectionnez Afficher le code pour ouvrir ScreenSaverForm.cs en mode Code.
  2. Faites défiler le code source jusqu'à ce que la méthode OnPaint soit visible, ou sélectionnez-la dans la liste déroulante dans le coin supérieur droit de la fenêtre en mode Code.

  3. Tapez le code suivant à la fin de la méthode :

    e.Graphics.DrawString("<Votre nom>Écran de veille C#", rssView.TitleFont, Brushes.Orange, new PointF(0, 0));

  4. Appuyez sur F5 pour réexécuter l'application.

Un titre est à présent affiché sur l'écran de veille.

Accès à la boîte de dialogue Options

Le Starter Kit Écran de veille comprend également une boîte de dialogue Options permettant aux utilisateurs de modifier les paramètres de l'écran de veille. Pour ouvrir la boîte de dialogue Options dans Visual Studio, vous devez envoyer un argument de ligne de commande directement à l'exécutable lors de son ouverture. Si vous utilisez Windows 98, sachez que les données que vous entrez dans la boîte de dialogue Options pourront être visualisées par toutes les personnes qui utilisent votre ordinateur.

Pour forcer l'apparition de la boîte de dialogue Options :

  1. Ouvrez l'Explorateur de solutions, puis double-cliquez sur le nœud Propriétés sous le projet ScreenSaver.
    Des options de configuration de votre projet s'affichent.
  2. Cliquez sur l'onglet Déboguer, puis recherchez la zone de texte Arguments de la ligne de commande.
  3. Tapez /c dans la zone de texte.
    Ceci indique à Visual C# d'exécuter l'écran de veille à l'aide de l'argument de ligne de commande /c.
  4. Appuyez sur F5 pour lancer l'application.
    La boîte de dialogue Options s'affiche.
  5. Découvrez les différentes options en sélectionnant un autre répertoire d'images d'arrière-plan ou en entrant un nouveau flux RSS.
  6. Cliquez sur OK pour fermer la boîte de dialogue.

Après avoir modifié les paramètres en mode d'options, vous pouvez réexécuter l'écran de veille en entrant /s comme argument de ligne de commande à la place de /c.

Installation de l'écran de veille sur votre ordinateur

Vous trouverez ci-dessous la procédure d'installation de l'application d'écran de veille sur votre ordinateur.

Pour installer l'écran de veille :

  1. Si ce n'est déjà fait, chargez le Starter Kit dans Visual C#.

  2. Dans le menu Fichier, cliquez sur Enregistrer tout.

  3. Si vous avez demandé à entrer un emplacement pour enregistrer le projet, sélectionnez un répertoire ou acceptez l'emplacement par défaut. Prenez note de l'emplacement auquel le projet est enregistré ; par défaut, il sera enregistré dans un répertoire sous \Mes documents\Visual Studio 2005\Projets.

  4. Dans le menu Générer, cliquez sur Générer la solution.
    Cette opération crée un fichier exécutable pour l'écran de veille.

  5. Ouvrez le répertoire spécifié à l'étape 3 qui contient le projet enregistré.

  6. Ouvrez le sous-répertoire bin\Debug. C'est à cet endroit que se situe la version debug de votre fichier exécutable. Son nom par défaut est ScreenSaver1.exe, mais il se peut que vous lui ayez attribué un autre nom lors de la création initiale du projet. Remplacez l'extension « .exe » de ce fichier par « .scr ».

    Remarque : Si les extensions de fichier de l'Explorateur Windows ne sont pas visibles, cliquez sur Options des dossiers sous le menu Outils, cliquez sur l'onglet Affichage et désactivez la case à cocher « Masquer les extensions des fichiers dont le type est connu ».

  7. Cliquez avec le bouton droit sur le fichier .scr, puis sélectionnez « Installer ».
    La boîte de dialogue Écran de veille Windows s'ouvre.

  8. Cliquez sur le bouton Aperçu de la boîte de dialogue Écran de veille.
    L'écran de veille est lancé. Appuyez sur une touche pour l'arrêter.

  9. Cliquez sur le bouton Paramètres dans la boîte de dialogue Écran de veille Windows pour visualiser les options de l'écran de veille. Essayez de changer l'image d'arrière-plan ou le flux RSS.

  10. Cliquez sur OK.
     

Vous venez d'installer votre propre écran de veille Windows.

Partage de votre version de l'écran de veille avec des amis

Maintenant que vous disposez d'une version personnalisée de l'écran de veille, vous pouvez la partager avec vos amis et votre famille. Notez que l'application d'écran de veille utilise la version 2.0 de .NET Framework, aussi, tout ordinateur qui l'exécute devra disposer de cette version de Framework. Microsoft .NET Framework est installé dans le cadre de Visual Studio, mais vous pouvez l'installer directement à partir du Centre pour développeurs Microsoft .NET Framework. Si vous prévoyez de partager votre écran de veille avec vos amis, vous devez les informer qu'ils devront installer la nouvelle version de .NET Framework.

Fonctionnement de l'écran de veille

Lorsque Windows active l'écran de veille à cause d'une période d'inactivité, l'exécutable est lancé (il porte le nom ScreenSaver1.scr à moins que vous lui ayez attribué un autre nom).

À moins que l'écran de veille ait été demandé pour ouvrir la boîte de dialogue Options, il crée un formulaire Windows Form plein écran, insère l'image d'arrière-plan par défaut et accède à Internet pour télécharger et afficher les données provenant des flux RSS. Deux minuteries mettent à jour la rubrique actuellement sélectionnée et modifient l'image d'arrière-plan.

En cas de déplacement de la souris ou de clic sur la souris à un moment ou un autre, ou si une touche est enfoncée, le programme se ferme immédiatement.

Fichiers projet de l'écran de veille

Le projet contient huit fichiers de code source C#.

Program.cs Contient la méthode Main() — l'emplacement auquel le programme commence l'exécution — et le code pour traiter les commutateurs de ligne de commande
OptionsForm.cs Contient le formulaire Windows Form qui affiche les options de paramètres. À partir de cette boîte de dialogue, vous pouvez sélectionner une image d'arrière-plan et un flux RSS à utiliser lorsque l'écran de veille est exécuté.
ScreenSaverForm.cs Contient le formulaire Windows Form principal de l'écran de veille. Les images d'arrière-plan et le flux RSS sélectionnés par l'utilisateur sont lus et affichés sur ce formulaire plein écran.
RSSFeed.cs Contient la classe représentant un flux RSS 2.0.
RssChannel.cs Contient la classe représentant un canal RSS. Chaque flux RSS comprend un seul canal. Il représente une collection d'articles.
RSSItem.cs Contient la classe représentant un élément RSS. Un canal RSS peut inclure n'importe quel nombre d'éléments RSS. Un élément RSS représente un article, une publication de blog ou un article de presse.
Item.cs Contient la classe représentant un élément, un objet doté d'un titre et d'une description.
ItemDescriptionView.cs Contient le code qui affiche la description d'un élément provenant d'un flux RSS. Il apparaît peu à peu en fondu dans le texte de l'élément, puis disparaît de nouveau en fondu.
ItemListView.cs Contient le code qui affiche le titre d'un élément provenant d'un flux RSS. Il assure le suivi d'un article actuellement sélectionné, qu'il peut faire défiler en avant ou en arrière.

Concepts

Interaction de Windows avec les écrans de veille

Un écran de veille Windows est un exécutable qui ouvre une fenêtre en plein écran présentant un affichage graphique changeant, tels que des étoiles ou une sélection de photos. Il est possible d'utiliser n'importe quel exécutable comme écran de veille Windows. Pour qu'un exécutable apparaisse dans la boîte de dialogue de sélection de l'écran de veille, il doit simplement être placé dans le répertoire Windows (le répertoire \Windows\System32 du lecteur système si vous utilisez Windows 98 ou Windows XP, ou le répertoire \WinNT\System32 si vous utilisez Windows NT ou Windows 2000).

Lorsque Windows exécute votre écran de veille, il le lance à l'aide de l'une des trois options de ligne de commande suivantes :

  • /s – Indique à l'écran de veille qu'il doit s'afficher.
  • /c – Indique à l'écran de veille qu'il doit ouvrir sa boîte de dialogue Paramètres.
  • /p – Indique à l'écran de veille qu'il doit afficher un petit aperçu.

Ce Starter Kit gère ces arguments dans le fichier source Program.cs. Lorsque vous observez le code dans ce fichier, vous constatez que l'argument /p n'est pas implémenté par ce Starter Kit.

Stockage des options à l'aide des concepteurs de ressources et de paramètres

Étant donné que le Starter Kit peut être exécuté dans deux modes différents, il doit disposer d'une méthode de stockage de ses paramètres entre les sessions, ce qu'il fait à l'aide du Concepteur de paramètres. Pour le visualiser, cliquez avec le bouton droit sur le nœud du projet ScreenSaver dans l'Explorateur de solutions, sélectionnez Propriétés, puis cliquez sur l'onglet Paramètres. Ce concepteur vous permet d'établir la liste de tous les paramètres que votre programme souhaite définir et d'y accéder à partir de votre code sous la forme :

Properties.Settings.Default.<NomParamètre>


De la même manière, le Concepteur de ressources permet de stocker le flux RSS et l'image d'arrière-plan par défaut. Pour le visualiser, cliquez avec le bouton droit sur le nœud du projet ScreenSaver dans l'Explorateur de solutions et sélectionnez Propriétés, puis cliquez sur l'onglet Ressources.

XML et RSS

Les flux RSS représentent un format XML pour la syndication de contenu sur le Web. Ils sont souvent utilisés par des sites pour publier leurs listes de nouveaux articles ou de publications de blog. Un flux RSS possède un nœud <rss> contenant un nœud <channel> (canal). Le nœud <channel> possède un ou plusieurs nœuds <item> (élément), un pour chaque article ou publication de blog dans le flux. Il contient également un élément <title> (titre), qui contient le titre du canal. Chaque <item> peut contenir un titre, un lien et une description. Voici un exemple très simple.

 

<rss version="2.0">

  <channel>

    <title>Mon flux</title>

    <description>Certains articles.</description>

    <link>http://www.microsoft.com/france/msdn/</link>     

    <item>

      <title>Article 1</title>

      <description>Résumé de l'article.</description>

      <link>http://msdn.microsoft.com/netframework</link>

    </item>

    <item>

      <title>Article 2</title>

      <description>Autre article.</description>

      <link>http://msdn.microsoft.com/vcsharp/</link>

    </item>

  </channel>

</rss>

Amélioration de l'écran de veille

Ajout d'une prise en charge du clavier

L'une des méthodes simples d'amélioration de l'écran de veille consiste à ajouter une prise en charge du clavier. Par exemple, vous pouvez autoriser l'utilisateur à sélectionner l'élément RSS actuel à l'aide des touches du curseur. La procédure à suivre est indiquée ci-dessous.

Pour ajouter une prise en charge du clavier :

  1. Dans l'Explorateur de solutions, cliquez avec le bouton droit sur ScreenSaverForm.cs et sélectionnez Concepteur de vues.
  2. Si la fenêtre Propriétés n'est pas encore affichée, ouvrez-la en cliquant sur le menu Affichage, puis en sélectionnant Fenêtre Propriétés.
  3. Dans la liste déroulante située en haut de la fenêtre Propriétés, sélectionnez ScreenSaverForm.
  4. Cliquez sur le bouton Événements (l'éclair) pour voir les événements que le formulaire peut déclencher.
  5. Recherchez l'événement KeyDown. Il aura une méthode, ScreenSaverForm_KeyDown, répertoriée comme gestionnaire pour l'événement.
  6. Double-cliquez sur l'événement KeyDown.
    Ceci ouvre le mode Code et vous amène à la méthode ScreenSaverForm_KeyDown.

  7. Remplacez la ligne Close(); par ce qui suit :

    switch(e.KeyCode)

                {

                    case Keys.Down:

                         // Faire pivoter les articles vers le bas.

                         rssView.NextArticle();

                         break;

                     case Keys.Up:

                         // Faire pivoter les articles vers le haut.

                         rssView.PreviousArticle();

                         break;

                     default:

                         Close();

                         break;

                 } 
  8. Enregistrez le fichier, puis appuyez sur F5 pour exécuter le projet.
  9. Lorsque l'écran de veille est en cours d'exécution, appuyez sur les touches Haut et Bas pour changer l'article actuellement sélectionné.

Suggestions complémentaires

L'application d'écran de veille est conçue pour être facilement modulable et personnalisable. Voici quelques idées pour cela :

  • Sélectionnez votre propre image par défaut intégrée :
    Remplacez l'image d'écran de veille par défaut par l'image de votre choix.
  • Affichez des informations météorologiques dans le coin supérieur droit :
    Utilisez un service Web tel que celui-ci pour récupérer des informations météorologiques, puis afficher des détails de la météo actuelle dans le coin supérieur droit de l'écran.
  • Générez votre propre écran de veille :
    Remplacez ScreenSaverForm par un autre Windows Form qui affiche un écran de veille totalement différent. Observez les écrans de veille Windows existants pour avoir des idées – les Étoiles et les Canalisations 3D ont déjà été utilisées maintes fois ; que pourriez-vous trouver de nouveau ? Le fichier Program.cs gère toutes les options de ligne de commande permettant de prendre en charge un écran de veille.
  • Plusieurs flux RSS :
    Développez la boîte de dialogue Options pour autoriser l'utilisateur à sélectionner un ensemble de flux RSS qui permettront de recueillir de nouveaux articles. Vous pouvez réaliser cela de plusieurs manières :
    1. La boîte de dialogue Options permet à l'utilisateur de sélectionner un fichier OPML contenant une liste de flux RSS,
    -ou-
    2. La boîte de dialogue Options permet à l'utilisateur de remplir une liste de flux RSS.
    Un flux aléatoire peut être choisi lorsque l'écran de veille se charge, puis un nouveau flux peut être choisi chaque fois que l'écran de veille arrête d'afficher le contenu d'un flux.
  • Terminez d'implémenter les classes RSS :
    Les classes RSS fournies dans ce Starter Kit ne gèrent pas l'intégralité de la spécification RSS 2.0. Le reste de la spécification peut être ajouté à ces classes. Les classes peuvent alors être « empaquetées » sous la forme d'une dll, puis réutilisées dans une autre application.
  • Réutilisez les classes RSS :
    Les classes RSS ne pouvaient pas être réutilisées dans une autre application. Vous avez la possibilité d'écrire votre agrégateur RSS ou de créer une application System Tray (barre d'état système) qui fait apparaître un article toutes les heures.
